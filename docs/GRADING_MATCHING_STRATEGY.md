# 分级匹配策略 - 优化说明

## 概述

为了提高匹配成功率同时保证准确性，我们实现了**两级分级匹配策略**：

1. **第一级（严格匹配）**：名称 + 年份 + 导演 + 地区
2. **第二级（宽松匹配）**：年份 + 导演 + 地区 + 名称模糊匹配

## 匹配流程

```
开始
  ↓
第一级：严格匹配（名称+年份+导演+地区）
  ↓
结果数量 = 1? → 返回该结果 ✅
  ↓
结果数量 > 1? → 返回 'multiple' ⚠️
  ↓
结果数量 = 0? → 进入第二级
  ↓
第二级：宽松匹配（年份+导演+地区，名称相似度>=0.5）
  ↓
结果数量 = 1? → 返回该结果 ✅
  ↓
结果数量 > 1? → 返回相似度最高的结果 ✅
  ↓
结果数量 = 0? → 返回 None ❌
```

## 详细策略

### 第一级：严格匹配

**要求**：所有字段都必须匹配
- **名称**: 精确匹配 OR 包含关系 OR 相似度 >= 0.7
- **年份**: 必须匹配
- **导演**: 必须匹配（如果提供）
- **地区**: 必须匹配（如果提供）

**返回值**：
- 1个结果 → 直接返回
- 多个结果 → 返回 `'multiple'`
- 0个结果 → 进入第二级

### 第二级：宽松匹配

**触发条件**：第一级结果为0

**要求**：
- **名称**: 相似度 >= 0.5（降低阈值）
- **年份**: 必须匹配
- **导演**: 必须匹配（如果提供）
- **地区**: 必须匹配（如果提供）

**返回值**：
- 1个结果 → 返回该结果
- 多个结果 → 返回相似度最高的那个
- 0个结果 → 返回 `None`

### 核心方法

```python
@staticmethod
def match_douban_search_results(search_results, target_name, target_year, 
                                 target_area='', target_director=''):
    """分级匹配策略"""
    
    # ========== 第一级：严格匹配 ==========
    strict_matched = []
    for result in search_results:
        if (name_match and year_match and area_match and director_match):
            strict_matched.append(result)
    
    if len(strict_matched) == 1:
        return strict_matched[0]
    elif len(strict_matched) > 1:
        return 'multiple'
    
    # ========== 第二级：宽松匹配 ==========
    if len(strict_matched) == 0:
        loose_matched = []
        for result in search_results:
            # 放宽名称要求：相似度 >= 0.5
            if (loose_name_match and year_match and area_match and director_match):
                loose_matched.append(result)
        
        if len(loose_matched) == 1:
            return loose_matched[0]
        elif len(loose_matched) > 1:
            # 返回相似度最高的
            return max(loose_matched, key=lambda x: similarity)
    
    return None
```

## 参数调整

如果需要调整匹配严格度：

```python

```

### 调整宽松模式名称阈值

```python
# data_processor.py 第490行左右
loose_name_match = similarity >= 0.5  # 提高→更严格，降低→更宽松
```

**建议**：
- 如果发现太多误匹配 → 提高阈值
- 如果发现太多漏匹配 → 降低阈值

## 优势

1. **准确性优先**：先尝试严格匹配，确保高质量结果
2. **容错性强**：严格匹配失败后自动降级到宽松匹配
3. **智能选择**：宽松模式下有多个结果时选择最相似的
4. **灵活性高**：可以根据实际情况调整阈值

## 总结

分级匹配策略通过**先严后松**的方式，在保证准确性的前提下最大化匹配成功率：

- **第一级**：确保高质量匹配（名称+年份+导演+地区）
- **第二级**：在第一级失败时提供备选方案（放宽名称要求）
- **智能决策**：多个结果时选择最优解

这个策略既解决了翻译差异问题，又避免了过度宽松的匹配导致的误匹配。
