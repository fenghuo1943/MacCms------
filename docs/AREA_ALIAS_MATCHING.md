# 地区别名匹配功能说明

## 功能概述

增强了地区匹配功能，支持常见地区别名的自动识别和匹配。例如："大陆" 和 "中国大陆" 可以成功匹配。

## 匹配策略

地区匹配采用三级策略：

1. **标准化精确匹配**：将目标地区和结果地区都标准化后进行比较
   - 例：`大陆` → `中国大陆` == `中国大陆` → `中国大陆` ✓

2. **原始值包含关系**：检查原始值是否存在包含关系
   - 例：`中国` in `中国大陆` ✓

3. **标准化后包含关系**：标准化后再检查包含关系
   - 例：`中国大陆` in `中国大陆, 美国` ✓

## 代码示例

### 基本使用

```python
from douban_fetcher.data_processor import DataProcessor

# 测试地区标准化
normalized = DataProcessor._normalize_area("大陆")
print(normalized)  # 输出: "中国大陆"

# 测试地区匹配
result = {'country': '中国大陆'}
match = DataProcessor._check_area_match('大陆', result)
print(match)  # 输出: True
```

### 在匹配流程中自动使用

地区匹配已在以下方法中自动启用：

1. `_check_area_match` - 基础地区匹配方法
2. `_refine_multiple_matches` - 二次确认中的地区匹配

无需手动调用，系统会自动处理别名。

## 扩展别名表

如需添加新的地区别名，修改 `_normalize_area` 方法中的 `area_aliases` 字典：

```python
area_aliases = {
    '现有别名': '标准名称',
    '新别名': '标准名称',  # 添加新的别名
    # ...
}
```

## 注意事项

1. **大小写敏感**：英文别名区分大小写（如 `USA` 而非 `usa`）
2. **空格处理**：自动去除首尾空格
3. **未知地区**：如果地区不在别名表中，保持原样进行匹配
4. **多地区支持**：支持逗号分隔的多地区（如 "中国大陆, 美国"）
