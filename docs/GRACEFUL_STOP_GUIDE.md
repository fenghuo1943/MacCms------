# 优雅停止指南

本文档介绍如何在使用 MacCMS 豆瓣评分自动获取工具时优雅地中途停止任务。

## 🎯 停止方法

### 方法一：Ctrl+C（推荐）⭐⭐⭐

在程序运行时，按下 **Ctrl+C** 即可优雅停止任务。

**特点：**
- ✅ 最简单直接
- ✅ 会等待当前视频处理完成后停止
- ✅ 自动保存进度和统计信息
- ✅ 不会丢失已处理的数据

**示例输出：**
```
正在处理视频...
  进度: 150 | 成功: 142 | 速度: 1.85/s | 预计剩余: 2小时30分钟

^C

⚠️  检测到停止信号 (Ctrl+C)
正在优雅停止任务...
任务已优雅停止！
总处理: 150 个
成功: 142 个 (94.7%)
```

---

### 方法二：Ctrl+Z ⭐⭐

在程序运行时，按下 **Ctrl+Z** 也可以停止任务。

**注意：**
- Windows 系统中 Ctrl+Z 可能被解释为 EOF（文件结束符）
- 在某些终端中可能不生效
- 建议优先使用 Ctrl+C

**示例输出：**
```
⚠️  检测到暂停/停止信号 (Ctrl+Z)
正在优雅停止任务...
任务已优雅停止！
```

---

### 方法三：创建停止文件 ⭐⭐⭐

在程序运行目录下创建一个名为 `stop.flag` 的空文件，程序检测到后会自动停止。

**适用场景：**
- 后台运行的程序
- 远程服务器
- 批处理脚本
- 无法直接交互的情况

**操作步骤：**

#### Windows PowerShell:
```powershell
New-Item -Path "stop.flag" -ItemType File
```

#### Windows CMD:
```cmd
type nul > stop.flag
```

#### 手动创建：
1. 在程序运行目录右键 → 新建 → 文本文档
2. 重命名为 `stop.flag`（注意扩展名）

**特点：**
- ✅ 适合远程服务器或后台运行的场景
- ✅ 可以提前创建，程序会在下一个检查点停止
- ✅ 停止后文件会自动删除
- ✅ 不会影响当前正在处理的视频

**示例：**
```bash
# 在另一个终端窗口执行
cd "e:\python\MacCms自动获取评分"
New-Item -Path "stop.flag" -ItemType File

# 程序会输出：
# 检测到停止文件: stop.flag
# 已删除停止文件
# 任务已优雅停止！
```

---

### 方法四：程序化停止（开发者）⭐

如果你需要在代码中控制停止，可以设置 `stop_requested` 标志：

```python
from douban_fetcher import DoubanScoreFetcher
import threading
import time

fetcher = DoubanScoreFetcher(db_config=...)

# 启动任务（在另一个线程中）
thread = threading.Thread(target=fetcher.run)
thread.start()

# 需要停止时
time.sleep(10)  # 运行10秒
fetcher.stop_requested = True  # 设置停止标志
thread.join()  # 等待线程结束
```

## 📋 停止行为说明

无论使用哪种停止方法，程序都会：

1. ✅ **完成当前视频的处理** - 不会中断正在进行的API请求
2. ✅ **保存所有进度** - 统计信息自动保存到 `fetch_stats_*.json`
3. ✅ **释放数据库锁** - 已锁定但未处理的视频会被释放
4. ✅ **生成最终报告** - 保存详细的任务报告
5. ✅ **保持数据一致性** - 已成功的数据会正确写入数据库

## 🔄 恢复任务

停止后可以随时重新启动任务，程序会自动：

- 从上次停止的位置继续
- 跳过已成功处理的视频
- 重新处理失败或超时的视频

只需再次运行：
```bash
python main.py
```

## ⚠️ 注意事项

### ❌ 不要强制终止程序！

以下操作可能导致数据不一致：
- 直接关闭命令行窗口
- 使用任务管理器强制结束进程
- 按 Ctrl+Break（在某些系统中）
- 使用 `kill -9` 命令

### ✅ 正确的做法：
- 使用 Ctrl+C（最推荐）
- 使用 Ctrl+Z
- 创建 stop.flag 文件
- 等待程序自然结束

## ❓ 常见问题

### Q1: 为什么按了 Ctrl+C 没有立即停止？
**A:** 程序会等待当前视频处理完成后再停止，这通常需要几秒钟。这是为了确保数据完整性。

### Q2: Ctrl+Z 不起作用怎么办？
**A:** Windows 系统中 Ctrl+Z 可能被解释为其他功能。建议使用 Ctrl+C 或创建 stop.flag 文件。

### Q3: stop.flag 文件没有被自动删除？
**A:** 可能是权限问题。可以手动删除它，下次运行时程序会重新检测。

### Q4: 停止后如何确认任务真的停止了？
**A:** 查看日志输出，应该会看到 "任务已优雅停止！" 的消息，并且程序会退出到命令行提示符。

### Q5: 可以在批处理脚本中使用吗？
**A:** 可以！在 `.bat` 文件中也可以按 Ctrl+C 来停止。

### Q6: 停止后会丢失数据吗？
**A:** 不会。程序会在停止前保存所有进度和统计信息。

## 🔧 技术细节

停止机制的实现原理：

### 1. 信号捕获
```python
signal.signal(signal.SIGINT, signal_handler)   # Ctrl+C
signal.signal(signal.SIGTERM, signal_handler)  # 终止信号
if hasattr(signal, 'SIGTSTP'):
    signal.signal(signal.SIGTSTP, signal_handler)  # Ctrl+Z (Unix)
```

### 2. 标志检查
在每个视频处理前后检查停止标志：
```python
if self.check_stop_condition():
    break
```

### 3. 文件监控
定期检查 `stop.flag` 文件是否存在：
```python
if os.path.exists(self.stop_flag_file):
    return True
```

### 4. 优雅退出
确保资源正确释放和数据持久化：
- 保存统计信息
- 生成任务报告
- 显示状态分布

这种设计保证了：
- ✅ 数据不丢失
- ✅ 状态一致
- ✅ 可以随时恢复
- ✅ 多设备并发安全
