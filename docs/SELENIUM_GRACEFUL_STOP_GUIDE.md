# Selenium 方案优雅停止指南

本文档介绍如何在使用 Selenium 方案获取豆瓣评分时优雅地中途停止任务。

## 🎯 停止方法

### 方法一：Ctrl+C（推荐）⭐⭐⭐

在程序运行时，按下 **Ctrl+C** 即可优雅停止任务。

**特点：**
- ✅ 最简单直接
- ✅ 会自动关闭浏览器
- ✅ 保存所有进度和统计信息
- ✅ 不会丢失已处理的数据

**示例输出：**
```
正在处理视频...
  进度: 50 | 成功: 45 | 速度: 0.85/s | 预计剩余: 3小时20分钟

^C

⚠️  检测到停止信号 (Ctrl+C)
正在优雅停止任务...
任务已优雅停止！
总处理: 50 个
成功: 45 个 (90.0%)
```

---

### 方法二：Ctrl+Z ⭐⭐

在程序运行时，按下 **Ctrl+Z** 也可以停止任务。

**注意：**
- Windows 系统中 Ctrl+Z 可能被解释为 EOF（文件结束符）
- 在某些终端中可能不生效
- 建议优先使用 Ctrl+C

---

### 方法三：创建停止文件 ⭐⭐⭐

在程序运行目录下创建一个名为 `stop.flag` 的空文件，程序检测到后会自动停止。

**适用场景：**
- 后台运行的程序
- 远程服务器
- 批处理脚本
- 无法直接交互的情况

**操作步骤：**

#### Windows PowerShell:
```powershell
New-Item -Path "stop.flag" -ItemType File
```

#### Windows CMD:
```cmd
type nul > stop.flag
```

**特点：**
- ✅ 适合远程服务器或后台运行的场景
- ✅ 可以提前创建，程序会在下一个检查点停止
- ✅ 停止后文件会自动删除
- ✅ 不会影响当前正在处理的视频
- ✅ 浏览器会自动关闭

---

### 方法四：程序化停止（开发者）⭐

如果你需要在代码中控制停止，可以设置 `stop_requested` 标志：

```python
from selenium_fetcher import SeleniumDoubanFetcher

fetcher = SeleniumDoubanFetcher(db_config=..., selenium_config={...})

# 启动任务（在另一个线程中）
import threading
thread = threading.Thread(target=fetcher.run)
thread.start()

# 需要停止时
time.sleep(10)  # 运行10秒
fetcher.stop_requested = True  # 设置停止标志
thread.join()  # 等待线程结束
# 浏览器会自动关闭
```

## 📋 停止行为说明

无论使用哪种停止方法，Selenium 方案都会：

1. ✅ **完成当前视频的处理** - 不会中断正在进行的网页操作
2. ✅ **自动关闭浏览器** - 确保没有残留的浏览器进程
3. ✅ **保存所有进度** - 统计信息自动保存到 `selenium_fetch_stats_*.json`
4. ✅ **释放数据库锁** - 已锁定但未处理的视频会被释放
5. ✅ **生成最终报告** - 保存详细的任务报告
6. ✅ **保持数据一致性** - 已成功的数据会正确写入数据库

## 🔄 恢复任务

停止后可以随时重新启动任务，程序会自动：

- 从上次停止的位置继续
- 跳过已成功处理的视频
- 重新打开浏览器
- 重新处理失败或超时的视频

只需再次运行：
```bash
python main_selenium.py
```

## ⚠️ 注意事项

### ❌ 不要强制终止程序！

以下操作可能导致问题：
- 直接关闭命令行窗口（浏览器可能未关闭）
- 使用任务管理器强制结束进程
- 按 Ctrl+Break（在某些系统中）
- 使用 `kill -9` 命令

**后果：**
- 浏览器进程可能残留
- 数据库锁可能未释放
- 统计数据可能丢失

### ✅ 正确的做法：
- 使用 Ctrl+C（最推荐）
- 使用 Ctrl+Z
- 创建 stop.flag 文件
- 等待程序自然结束（浏览器会自动关闭）

## ❓ 常见问题

### Q1: 为什么按了 Ctrl+C 没有立即停止？
**A:** 程序会等待当前视频处理完成后再停止，这通常需要几秒到几十秒（取决于网页加载速度）。这是为了确保数据完整性和正确关闭浏览器。

### Q2: 停止后浏览器还在运行怎么办？
**A:** 正常情况下浏览器会自动关闭。如果浏览器仍然运行，可以：
1. 手动关闭浏览器窗口
2. 或在任务管理器中结束浏览器进程
3. 下次运行时会自动重新打开浏览器

### Q3: Ctrl+Z 不起作用怎么办？
**A:** Windows 系统中 Ctrl+Z 可能被解释为其他功能。建议使用 Ctrl+C 或创建 stop.flag 文件。

### Q4: stop.flag 文件没有被自动删除？
**A:** 可能是权限问题。可以手动删除它，下次运行时程序会重新检测。

### Q5: 停止后如何确认任务真的停止了？
**A:** 查看日志输出，应该会看到 "任务已优雅停止！" 的消息，并且程序会退出到命令行提示符，浏览器窗口也会关闭。

### Q6: 可以在批处理脚本中使用吗？
**A:** 可以！在 `.bat` 文件中也可以按 Ctrl+C 来停止，浏览器会自动关闭。

### Q7: 停止后会丢失数据吗？
**A:** 不会。程序会在停止前保存所有进度和统计信息。

## 💡 最佳实践

1. **定期检查进度** - 观察日志输出，了解处理进度
2. **预估剩余时间** - 日志会显示预计剩余时间
3. **选择合适时机停止** - 在批次之间停止更快
4. **保留日志文件** - 方便后续分析和问题排查
5. **优先使用 Ctrl+C** - 最简单可靠的停止方式
6. **确认浏览器已关闭** - 停止后检查是否有残留的浏览器进程

## 🔧 技术细节

停止机制的实现原理：

### 1. 信号捕获
```python
signal.signal(signal.SIGINT, signal_handler)   # Ctrl+C
signal.signal(signal.SIGTERM, signal_handler)  # 终止信号
if hasattr(signal, 'SIGTSTP'):
    signal.signal(signal.SIGTSTP, signal_handler)  # Ctrl+Z (Unix)
```

### 2. 标志检查
在每个视频处理前后检查停止标志：
```python
if self.check_stop_condition():
    break
```

### 3. 文件监控
定期检查 `stop.flag` 文件是否存在：
```python
if os.path.exists(self.stop_flag_file):
    return True
```

### 4. 优雅退出流程
```
收到停止信号
    ↓
设置停止标志
    ↓
完成当前视频处理
    ↓
保存统计信息
    ↓
释放数据库锁
    ↓
关闭浏览器（finally块保证执行）
    ↓
生成任务报告
    ↓
显示最终统计
    ↓
程序退出
```

### 5. 浏览器自动关闭
使用 `try-finally` 确保浏览器一定会被关闭：
```python
try:
    while True:
        # 主循环
        ...
finally:
    # 确保浏览器关闭
    self.browser_manager.quit_driver()
```

这种设计保证了：
- ✅ 数据不丢失
- ✅ 状态一致
- ✅ 浏览器正确关闭
- ✅ 可以随时恢复
- ✅ 多设备并发安全
